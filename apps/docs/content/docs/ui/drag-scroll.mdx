---
title: DragScroll
description: A headless component suite for adding mouse drag-to-scroll functionality to containers with optional navigation buttons and state subscription.
---

The `DragScroll` component suite provides mouse drag-to-scroll behavior for horizontal or vertical scrollable containers. It follows a compound component pattern that separates logic (Root) from the scroller element (Container), allowing for flexible layouts where navigation controls can stay outside the scroll area.

## Installation

```package-install
@zayne-labs/ui-react
```

## Basic Usage

The recommended way to use DragScroll is via the compound components, which manage context and state internally.

```tsx
import { DragScroll } from "@zayne-labs/ui-react/ui/drag-scroll";

function HorizontalScroller() {
	return (
		<DragScroll.Root>
			<DragScroll.Prev className="absolute top-1/2 left-0 -translate-y-1/2">←</DragScroll.Prev>

			<DragScroll.Container className="flex gap-4 overflow-x-auto p-10">
				{[1, 2, 3, 4, 5].map((item) => (
					<DragScroll.Item
						key={item}
						className="flex h-40 w-64 shrink-0 items-center justify-center rounded-xl border border-zu-primary/20 bg-zu-primary/10"
					>
						Item {item}
					</DragScroll.Item>
				))}
			</DragScroll.Container>

			<DragScroll.Next className="absolute top-1/2 right-0 -translate-y-1/2">→</DragScroll.Next>
		</DragScroll.Root>
	);
}
```

## Hook Usage (Advanced)

If you need fine-grained control or want to apply props to custom components without using the compound components, you can use the `useDragScroll` hook.

```tsx
import { useDragScroll } from "@zayne-labs/ui-react/ui/drag-scroll";

function CustomScroller() {
	const { propGetters } = useDragScroll();

	return (
		<div {...propGetters.getContainerProps()} className="flex gap-4 overflow-x-auto">
			{/* ... items */}
		</div>
	);
}
```

## Anatomy

```tsx
<DragScroll.Root>
	<DragScroll.Prev />

	<DragScroll.Container>
		<DragScroll.Item />

```

## Return Values (from `useDragScroll`)

<Accordions>
	<Accordion title="propGetters">
		Functions that return a collection of props to be spread onto your elements: - `getContainerProps`:
		Props for the scrollable container. - `getItemProps`: Props for each scrollable item. -
		`getPrevButtonProps`: Props for the previous navigation button. - `getNextButtonProps`: Props for the
		next navigation button.
	</Accordion>
	<Accordion title="containerRef">
		A standard React ref object containing the scrollable container element.
	</Accordion>
	<Accordion title="storeApi">The raw Zustand-style store API for advanced state management.</Accordion>
	<Accordion title="useDragScrollStore">
		A hook-based selector for subscribing to specific pieces of state.
	</Accordion>

## Styling

The following data attributes are automatically applied to the components for scoped styling:

| Component   | Attribute       | Value               | Description                                       |
| :---------- | :-------------- | :------------------ | :------------------------------------------------ |
| `Container` | `data-dragging` | `"true" \| "false"` | Whether the container is currently being dragged. |
| `Container` | `data-scope`    | `"drag-scroll"`     | Always set to `"drag-scroll"`.                    |
| `Container` | `data-part`     | `"container"`       | Identifies the container part.                    |
| `Prev`      | `data-disabled` | `"true" \| "false"` | Active when the button is logicaly disabled.      |
| `Next`      | `data-disabled` | `"true" \| "false"` | Active when the button is logicaly disabled.      |

```css title="styles.css"
[data-scope="drag-scroll"][data-part="container"][data-dragging="true"] {
	cursor: grabbing;
	user-select: none;
}

[data-scope="drag-scroll"][data-part="prev"][data-disabled="true"] {
	opacity: 0.5;
	pointer-events: none;
}
```
